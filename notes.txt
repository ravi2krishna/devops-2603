Class - 1
==============

    -> What Is DevOps ? 

        -> DevOps is a set of "PRACTICES" that works to "AUTOMATE" and "INTEGRATE"
        the process between Software Development (Dev) and IT Operations(Ops) Teams.

    -> Development(Dev) Team

        -> To "Build Applications", we need roles like 

            -> Software Architects 
            -> Softwares Developers 
            -> Softwares Testers 
            -> etc 

    -> IT Operations(Ops) Team

        -> To "Build Infrastructure" & "Maintain Applications", we need roles like 

            -> System Admin
            -> Network Admin 
            -> Cloud Admin 
            -> etc 


    -> DevOps Engineer 

        -> DevOps Engineers will implement DevOps "PRACTICES" Using "TOOLS", 
        so that organizations can Build, Test & Deploy (Delivery Line) 
        Softwares More "FASTER" & "RELIABLE".


    -> DevOps "PRACTICES"                               ==>            DevOps "TOOLS"

        -> Version Control System (VCS)                 ==>            Git 

            -> For Tracking Code 

            -> Happens Inside Developer Laptop - Local To Individual Developer 

        
        -> Source Code Management (SCM)                 ==>            GitHub

            -> For Storing Multiple Versions Of Code  

            -> Happens Inside Organization - Central To All Developers 

        
        -> Containerization                             ==>            Docker

            -> For Getting Acceleration i.e Making FASTER 

        
        -> Orchestration                                ==>            Kubernetes 

            -> For Getting Reliability 

            -> RELIABLE means running Applications Without Downtime (Run 24 X 7) 
            i.e Production Environments need Reliability


        -> CI / CD                                      ==>            Jenkins 

            -> Continuous Integration & Continuous Deployment 

            -> For Automating Repetitive Tasks / Works

    -> Version Control Systems (VCS)

        -> Version control / Revision control / Source control
        is a process that helps to "keep track of changes" made to 
        files (computer programs, web site files, documents etc)

        -> https://www.sirhow.com/uploads/2023/07/check-gpay-transaction-history-step-4.jpg

        -> Advantage Of VCS is Revert Changes To Previous Working Versions, 
        if there are issues in new version

            -> https://notepad-plus-plus.org/downloads/

            -> Transaction ID's == Versions == Commit ID's 

    -> To Implement Version Control System, We need "Git" Software / Tool 

        -> Git Helps in Tracking Code / Changes In Your Code 

        -> Git Tracks Code in your Laptop

        -> Git will generate commit id for every change we make 
        (like every transaction we make in UPI)


    -> All the transactions made will be stored in Server, Similarly we store 
    all our commit id's in Source Code Management(SCM) like a server 

        -> Mobile Phone -> Google Pay -> Bank Server (Transaction ID's)

        -> Laptop -> Git -> GitHub (Commit ID's)

    -> GitHub 

        -> GitHub provides a platform to store code and it's versions 

        -> GitHub Stores all versions of code in Centralized location (github.com)
